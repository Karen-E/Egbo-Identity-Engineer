federation, token-based authentication flows, and OAuth 2.0 
authorization — built using a live Okta environment.

## Why I built this
SAML handles legacy enterprise SSO well, but OIDC is what modern 
applications use. Building hands-on experience with both protocols 
gives me the ability to work across the full spectrum of enterprise 
identity integrations — from legacy on-premise apps using SAML to 
cloud-native applications using OIDC and OAuth 2.0.
